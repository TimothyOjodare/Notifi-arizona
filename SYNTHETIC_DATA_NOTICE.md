# Synthetic Data Notice

**Every patient, animal, provider, and report in this repository is entirely synthetic.**

## Methodology

- All data is generated by deterministic Python scripts in `src/data_generation/`
  using `random.Random(seed=42)`. Re-running the pipeline produces the same
  dataset, bit-for-bit.
- No real patient or animal was sampled, anonymized, or de-identified for this
  project. There is no underlying real dataset.
- Names, addresses, dates of birth, lab values, exposure histories, and
  reporting events are all procedurally generated to be epidemiologically
  plausible against published Arizona disease patterns.
- The 72-disease reportable database (`data/reference/reportable_diseases_us.json`)
  is a public-data product compiled from:
    - Arizona Administrative Code R9-6-202 (effective June 2025)
    - CDC National Notifiable Diseases Surveillance System
    - USDA APHIS National List of Reportable Animal Diseases
    - AZ Department of Agriculture state veterinarian's reportable list
    - AZ Game and Fish Department wildlife disease guidance
  This database is real public-policy data. The patient and animal cases that
  reference it are synthetic.

## Names of real institutions and agencies

This repository references real Arizona institutions and government agencies
by name — Tucson Medical Center, Banner Health, IHS Whiteriver, Arizona
Department of Health Services, Apache Tribal Health Authority, Navajo Nation
Department of Health, USDA APHIS, AZ Department of Agriculture, AZ Game and
Fish Department, CDC NNDSS. **None of these references represent real
agreements, partnerships, integrations, or endorsements.** They appear because
the project is geographically specific to Arizona and the institutional
landscape is a public fact.

Production deployment of this system would require Memoranda of Understanding
with each named agency, real ELR endpoint registration with ADHS-MEDSIS, real
NEDSS submission credentials with CDC, real VSPS account provisioning with
USDA APHIS, and IRB-approved data-flow protocols with each tribal health
authority. None of these exist for this MVP.

## Pre-deployment validation pathway

The MVP demonstrates the workflow against simulated agency endpoints. The
pathway from this prototype to a real-data deployment:

  1. Real-endpoint registration with ADHS-MEDSIS, with parallel-processing
     window where reports route to both NotifiAZ and the existing fax workflow
     so the agency can compare outputs.
  2. Pilot deployment at one clinical site (TMC is the natural first site)
     with a defined scope of two diseases (Coccidioidomycosis and
     Salmonellosis) for the first three months.
  3. Tribal IRB partnership before any tribal-health-authority routing goes
     live in production.
  4. Audit review at month 3, 6, and 12, comparing report completeness,
     time-to-submit, and callback rate against historical baseline.

None of these are in scope for the capstone.
