# For Evaluators — Read This First

This repository contains the **graded submission** for the Digital Public Health capstone:
**NotifiAZ** — a focused single-thesis system for reportable disease reporting.

## Two-repository project structure

This project's full intellectual journey is preserved across **two GitHub repositories**:

| Repository | What it is | Status |
|---|---|---|
| **`notifi-az`** (this repo) | Final capstone submission. Focused thesis: reportable disease reporting in 4 minutes, multi-agency, bidirectional. | **GRADED SUBMISSION** |
| **`onehealth-record-v1`** ([link](https://github.com/YOUR_USERNAME/onehealth-record-v1)) | Original prototype before scope pivot. Broader One-Health EHR with federated learning, swarm learning, and a deep-learning clinical risk model. | Archived — for context only |

## Why two repositories?

After committee feedback that the original prototype was solving too many problems
at once, the project was narrowed to a single defensible thesis. NotifiAZ is the
result of that pivot.

The earlier work is preserved in a separate repository so it remains visible
(demonstrating response to feedback and showing the intellectual journey) but
unambiguously separate from the grading submission.

## What is being graded

**Only this repository (`notifi-az`) is the grading submission.** The companion
repository is included for context — to show what was tried, what was scoped out,
and why. It is not part of the evaluation.
